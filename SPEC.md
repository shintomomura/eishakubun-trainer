# 瞬間英作文トレーナー 仕様書

**バージョン**: 1.0  
**作成日**: 2026-05-08  
**作成**: ソフトウェア部 山本翼  
**承認待ち**: CEO（友村晋） / COO 坂本龍馬

---

## 1. 目的

CEOがブラウザで「瞬間英作文（日本語→英語の即興英作文）」を**ランダム順**にトレーニングし、**できなかった問題だけ**を後から繰り返し復習できる学習用Webアプリ。

## 2. 出題データ

- **ソース**: CEO所有のGoogleスプレッドシート（[元URL](https://docs.google.com/spreadsheets/d/1PQknGfb_XYxUMydvWPgD3dBXi-bZqZIRvoZKqzbyElA/)、**閲覧公開**設定）
- **取り込み方式**: 起動時にスプレッドシートのCSVを直接 `fetch` する **ライブ取得方式**
  - スプレッドシートに行を **追加・編集・削除** すると、次回起動時に自動反映
  - 失敗時はリポジトリ同梱の `problems.json` にフォールバック
- **ID**: 内容ハッシュ（`ja + en` のFNV-1a 32bit）。行の入れ替え・削除があっても同じ問題は同じIDを保つため、「できなかった」リストの整合性が壊れない
- **問題数**: 公開時点で420問（日本語と英語の対訳が両方入っている行のみ採用、空の行2件は除外）

## 4. 機能要件

### 4.1 通常モード（デフォルト）

1. 起動時に全420問をシャッフルしてキューを作る
2. キューの先頭問題の **日本語** を画面に表示
3. ユーザーが「**答えを見る**」を押すと、英語の正解を表示する
   - **重要**: 自動で答えは出ない。必ずユーザーが押すまで英語は隠したまま
4. 答えを見た後、ユーザーが ○ / × を自己採点でクリック
   - **○ できた** → 正解カウント+1、もし「できなかった」リストに入っていたら除外、次の問題へ
   - **× できなかった** → 不正解カウント+1、「できなかった」リストに追加（重複なし）、次の問題へ
5. 「スキップ」を押すと、その問題はキューの末尾に回される（後でまた出題される）
6. 全問終わったら「全問終了！」画面を表示し、できなかった問題が残っていれば「復習モードへ」ボタンを表示

### 4.2 復習モード

1. 「できなかった」リスト（LocalStorage保存）に登録されている問題だけをシャッフルして出題
2. 操作は通常モードと同じ
3. 復習で ○ を付けた問題は「できなかった」リストから除外される
4. 全部 ○ になるとリストが空になり、「全問クリア！」画面を表示

### 4.3 永続化（LocalStorage）

ブラウザを閉じても次の状態が残る。

- `mode`: 現在のモード（`all` / `review`）
- `queue`: 残りの問題ID配列
- `okCount` / `ngCount`: 現在のセッションの正誤カウント
- `wrongIds`: 「できなかった」問題ID配列（**復習対象**）

LocalStorageキー: `eishakubun:v1`

### 4.4 リセット

- **「最初から」ボタン**: 現在のモードをシャッフルし直して再スタート（`wrongIds` は維持）
- **「学習データをリセット」ボタン**（フッタ）: `wrongIds` も含めて全消去

### 4.5 操作（キーボードショートカット）

| キー | 動作 |
|---|---|
| Space / Enter | 答えを見る |
| 1 または O | ○ できた |
| 2 または X | × できなかった |
| S | スキップ |
| Ctrl/⌘ + Enter（入力欄内）| 答えを見る → ○ |

## 5. 非機能要件

| 項目 | 仕様 |
|---|---|
| デバイス | PC・スマホ両対応（レスポンシブ） |
| テーマ | OS設定に応じてダーク/ライト自動切替 |
| 通信 | 起動時に `problems.json` を1回読むだけ。以降オフライン動作可 |
| 依存ライブラリ | なし（Vanilla JS／フレームワークなし／ビルド不要） |
| ホスティング | GitHub Pages（静的サイト） |
| ブラウザ | Chrome / Safari / Edge / Firefox の最新版 |

## 6. やらないこと（明示的な非対応）

- ❌ **タイマーで勝手に答えを表示する機能**（CEO要望により明示的に不採用）
- ❌ 自動採点（英文の正誤を機械判定するロジック。発音違い・語順違い等で誤判定するリスクが高いため採用せず、自己採点に統一）
- ❌ ユーザー登録・ログイン
- ❌ サーバー側の進捗保存（端末ごとのLocalStorageに留める）
- ❌ 音声・発音機能
- ❌ 別言語の問題集対応（必要になったら `problems.json` を差し替えで対応）
